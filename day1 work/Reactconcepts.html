React Concepts Guide

A comprehensive overview of fundamental React concepts and terminology

What is React?

React is a JavaScript library used to build user interfaces, especially for Single Page Applications (SPAs). It provides efficient ways to create interactive and dynamic web applications.

JavaScript vs JSX

JavaScript (JS)

JavaScript is a programming language used to make websites interactive and dynamic. It runs in the browser and can:

  • Change HTML content
  • Handle user events (clicks, typing)
  • Fetch data from servers
  • Build full web applications

JSX (JavaScript XML)

JSX is a syntax used in React that allows us to write HTML-like code inside JavaScript. It makes UI code easier to read and understand.

Example: 
const element = <h1>Hello World</h1>;

What is main.jsx? Why do we use it?

main.jsx is the entry point file in a React project (especially in Vite). It is responsible for:

  • Starting the React application
  • Rendering the root component (App.jsx)
  • Connecting React with the HTML div id="root"
Example: 
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App.jsx";

ReactDOM.createRoot(document.getElementById("root")).render(
  <App />
);

Summary: main.jsx → starts the React app

What is Babel Compiler?

Babel is a JavaScript compiler that converts:

  • JSX → JavaScript
  • Modern JavaScript → Browser-compatible JavaScript
Example - JSX Code: 
const element = <h1>Hello</h1>;
Babel converts it into: 
const element = React.createElement("h1", null, "Hello");

This conversion allows browsers to understand JSX code properly.

Virtual DOM vs Real DOM

Real DOM

The actual HTML structure in the browser.

Example: 
<h1>Hello</h1>

Problems:

  • Updating the real DOM is slow
  • Each change re-renders the entire page

Virtual DOM

A lightweight copy of the real DOM used by React.

Process:

  1. React creates a Virtual DOM
  2. When state changes → React compares old and new Virtual DOM (diffing)
  3. Only the changed part is updated in the real DOM

Result: This makes React very fast.

Export Default vs Named Exports

Export Default

A file can have only one default export.

Example: 
export default function App() {
  return <h1>Hello</h1>;
}

Named Export

You can export multiple things from a file.

Example: 
export const name = "Vijay";
export const age = 22;

How to Import Export Default and Named Exports?

Default Export Import

File:

export default App;

Import (No curly braces needed):

import App from "./App";

Named Export Import

Example:

import { name, age } from "./file";
